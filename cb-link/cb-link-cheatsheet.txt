CB-LINK CHEAT SHEET
===================

WORKFLOW (AMD HOST / FEDORA)
----------------------------
1. Host: cbe (extend) or cbm (mirror)
2. Client:
   - Chromebook (WiFi): cbv (windowed) or cbcm (scaled/fullscreen)
   - Tablet (WiFi): use VNC app to connect to host (host.local:5900 or IP)
   - Tablet (USB): cbmr (mirror + tablet) or cbt (tablet viewer only)
3. Host: cbs (stop server)

EXTEND MODE (2ND SCREEN)
-----------------------
Host: cbe
CB:   cbc / cbv (windowed)
CB:   cbcf (safe fullscreen via ssvnc)
CB:   cbtf (TigerVNC fullscreen legacy)

MIRROR MODE (VIEW-ONLY CLONE)
-----------------------------
Host: cbm
CB:   cbcm (auto-scaled, recommended - scales to fit CB screen)
CB:   cbcf (safe fullscreen via ssvnc)
CB:   cbv  (windowed TigerVNC)
NOTE: TigerVNC fullscreen (cbtf or in-app mod+f) can trigger resize
      requests that cause gray screen in mirror mode. Prefer cbcf/cbcm.

SCREEN ROTATION (HOST)
----------------------
Host: mod+o  - cycle eDP-1 through 0/90/180/270
After rotating: re-run cbm, then reconnect with cbcm
wl-mirror loses source on transform change; no auto-recovery yet.

TABLET (USB ADB REVERSE)
------------------------
Host: cbt  (open tablet viewer via ADB reverse)
Host: cbts (remove ADB reverse)
Host: cbmr (mirror + tablet)

TOGGLE / STATUS
---------------
Host: cbtog (toggle extend/mirror)
Host: cbst  (status)

FIREWALL (HOST)
---------------
Host: SF   (stop firewalld - allows VNC)
Host: sF   (start firewalld)
Host: zof  (open port 5900 only)

DISCONNECT
----------
Host: cbs  (stop server)
CB:   cbcd (disconnect client)
